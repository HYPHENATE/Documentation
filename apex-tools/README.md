# ApexTools

[![Install (Production)](https://img.shields.io/badge/Install-Production-blue)](https://login.salesforce.com/packaging/installPackage.apexp?p0=04tQB0000029eOTYAY)
[![Install (Sandbox)](https://img.shields.io/badge/Install-Sandbox-lightblue)](https://test.salesforce.com/packaging/installPackage.apexp?p0=04tQB0000029eOTYAY)

This package provides a reusable toolbox of Apex utilities and metadata-driven frameworks for Salesforce development. It is designed to reduce repeated boilerplate, improve consistency, and give teams a set of building blocks for common platform patterns such as trigger orchestration, logging, record sharing, test data generation, sandbox seeding, email automation, and dynamic querying.

Many Salesforce teams end up rebuilding the same technical foundations across projects: trigger frameworks, error logging, helper libraries, test factories, sharing logic, and admin-configurable automation support. While each of these can be built separately, maintaining them across multiple orgs and packages can become time-consuming and inconsistent.

This package brings those patterns together into a single unlocked package so they can be reused across implementations. It includes a metadata-driven trigger handler framework, configurable Apex logging, dynamic Apex sharing utilities, test record generation tools, sandbox post-copy data generation support, email actions for Flow, JSON parsing helpers, query builders, describe caching, and general-purpose Apex utility classes.

The goal is to give developers and technical teams a practical foundation for building scalable Salesforce solutions more quickly, while keeping common infrastructure patterns standardised, configurable, and easier to maintain.
