# v7.2.2

## Send Email Apex Action
 
There is a new version of the flow action Send Email Apex Action.  The current version will not be affected, so any client org. that has the current version of Apex Tools (or lower) will be unaffected by this change.
The original version works as it has always done, but it has a new description so it's easy to spot in the Apex Action selector in your flow builder:
Label : "Send Email Apex Action"
Description : "Defaults treatTargetObjectAsRecipient to true - ignores user input"
 
The new version of the Apex Action:
Label : "Send Email Apex Action - Version 2"
Description : "Respects user input of treatTargetObjectAsRecipient - sets value on email message as set"
 
As you can see, the problem with the original version was that the variable "treatTargetObjectAsRecipient" was not processed correctly in the code, and in fact always set the value to true.  The new version sets this variable correctly.
 
## Dynamic Share Creation
 
We wrote some code to handle sharing records via custom metadata configuration.  After some extensive testing from Sophie Morrison we realised that standard objects were not being processed correctly.  The code has been updated so that standard objects can now be shared using this code & custom metadata.
