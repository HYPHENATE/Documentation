# Tagging Examples

This page provides practical examples of how tags can be used within the **SimpleTags** framework.

While the tagging system is flexible and can be adapted to many use cases, these examples demonstrate common ways organisations use tags to classify records and improve reporting.

Many grantmaking organisations use tagging to support:

- thematic reporting on funding programmes
- identifying suitable expert reviewers
- categorising grant applications
- analysing funding distribution across strategic priorities

---

# Grant Management Examples

## Thematic Reporting on Funding Programmes

Tags can be used to classify **funding requests or grant applications** according to strategic themes.

These tags can be applied:

- during the **application process**
- during **programme officer review**
- during **grant approval workflows**

### Example

| Funding Request | Tags |
|------|------|
| Youth Education Programme Grant | Education, Youth Development |
| Community Mental Health Initiative | Health, Community Support |
| Climate Innovation Fund | Climate Action, Sustainability |

### Benefits

- Enables reporting by funding theme
- Supports portfolio analysis
- Helps track investment across strategic priorities

Example report:

| Report | Description |
|------|------|
| Funding Requests by Theme | Shows how many grants fall into each thematic area |
| Funding Amount by Programme Area | Total funding allocated by strategic theme |

---

## Expert Reviewer Matching

Tags can also be used to classify **reviewers or subject-matter experts** based on their expertise.

These tags allow programme teams to quickly identify appropriate reviewers for a grant application.

### Example

| Reviewer | Tags |
|------|------|
| Dr. Sarah Williams | Health, Mental Health |
| Prof. James Clarke | Climate Science, Sustainability |
| Dr. Emma Taylor | Education, Youth Development |

### Matching Workflow

1. A funding application is tagged with relevant themes.
2. Reviewers with matching expertise tags are identified.
3. The programme team assigns reviewers based on those tags.

Example:

| Grant Application | Tags |
|------|------|
| Community Mental Health Pilot | Health, Mental Health |

Possible reviewers:

| Reviewer | Expertise Tags |
|------|------|
| Dr. Sarah Williams | Health, Mental Health |
| Dr. Robert Green | Public Health |

### Benefits

- Improves reviewer matching
- Reduces manual search effort
- Helps maintain consistent review quality

A well-structured tagging system might look like this:

| Category | Example Tags |
|------|------|
| Funding Theme | Health, Education, Climate Action |
| Strategic Priority | Youth Development, Sustainability |
| Expertise Area | Mental Health, Climate Science |
| Programme Area | Community Development, Rural Support |

---

# Example Tag Application Workflow

A typical workflow may look like this:

1. A grant application is submitted.
2. During application review, the programme team applies thematic tags.
3. Expert reviewers with matching expertise tags are identified.
4. The grant is assessed and approved.
5. Reports analyse funding distribution by theme.

---

# When to Create New Tags

Consider creating a new tag when:

- a new funding theme emerges
- a programme area expands
- reporting requires additional classification

Before creating a new tag, always:

1. Check whether an appropriate tag already exists.
2. Ensure the tag belongs in the correct category.
3. Follow the naming guidelines outlined in **Tag Governance**.
