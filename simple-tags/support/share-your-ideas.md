# Share Your Ideas

SimpleTags is designed to evolve with the needs of the organisations that use it.

If you have suggestions for improvements, new features, or enhancements, we encourage you to share them.

> **Submit an idea:**  
> JIRA Link: `<<ADD YOUR JIRA IDEAS LINK HERE>>`

---

# Types of Ideas We Welcome

We welcome ideas related to:

- new tagging capabilities
- improvements to components
- reporting enhancements
- improvements to tag governance
- improvements to documentation
- integration with other Salesforce features

---

# Suggestion Template

When submitting an idea, please provide the following information.

| Information | Description |
|------|------|
| Idea Title | A short summary of the idea |
| Problem Statement | What problem does this idea solve? |
| Proposed Solution | How you think the feature should work |
| Example Use Case | A real scenario where this would be useful |
| Expected Benefit | What value this provides to users |
| Priority | Nice to have, Useful, High impact |

---

# Example Idea

| Field | Example |
|------|------|
| Idea Title | Tag Usage Dashboard |
| Problem Statement | Administrators cannot easily see which tags are most frequently used |
| Proposed Solution | Provide a dashboard showing tag usage across objects |
| Example Use Case | Programme teams want to see which thematic areas appear most often |
| Expected Benefit | Better insight into tagging and programme trends |

---

# How Ideas Are Evaluated

Ideas are typically reviewed based on:

- overall usefulness
- alignment with the SimpleTags architecture
- implementation complexity
- impact on existing functionality

Some ideas may be added to the product roadmap, while others may be implemented as configuration guidance or best practices.
