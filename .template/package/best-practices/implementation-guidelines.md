# Implementation Guidelines

## Recommended Setup

Describe the preferred way to roll out {{PACKAGE_NAME}}, including which nonprofit teams should be involved and which business process should be agreed before configuration starts.

## Best Practices

- Tie configuration decisions back to a real nonprofit process rather than implementing features in isolation.
- Use examples that show how fundraising, membership, grant, programme, or outcome teams will work with the package.
- Define naming, ownership, and reporting rules early so data stays understandable across teams.

## Avoid

- Generic examples with no business context.
- Configuring the package before agreeing what operational question or reporting need it is meant to support.
