# Implementation Guidelines

Keep messages short and action-oriented. The package works best when each message helps a user decide what to do next rather than trying to replace full training material.

Use filters to keep guidance specific. If every record shows the same large message list, users are more likely to ignore the component.

Separate editorial ownership from broad visibility. Give a small group the `Help Message Editor` permission set and use `Help Message Viewer` for the wider user base.

Review draft and published content regularly. Because guidance can change as processes evolve, it helps to treat messages as operational content that needs maintenance rather than one-time setup.
