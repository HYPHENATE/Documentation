# User Guide

## Overview

Dynamic Checks is designed to appear on a Lightning record page and show users the checklist categories and items that apply to the current record. Users complete checks as part of normal operational work rather than leaving the record to follow a separate checklist process.

## Typical Workflow

1. A user opens a record that has the Dynamic Checks component on its Lightning page.
2. The component loads the categories and checks that apply to that record.
3. The user expands categories, completes standard checks, selects answers for custom checks, and adds comments where permitted.
4. The checklist refreshes automatically so the team can see up-to-date progress on the record.

## Tips

- Use category names that reflect real operational stages, such as "Eligibility", "Approval Readiness", or "Case Handover".
- Keep check wording short and action-oriented.
- Use filtered checklist headers where you want to avoid showing irrelevant checks.
- Treat custom-answer checks as controlled prompts rather than general-purpose data capture.
