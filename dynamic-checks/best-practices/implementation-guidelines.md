# Implementation Guidelines

## Recommended Setup

Start with one business process and one target object before expanding usage across the organisation. Agree the operational purpose of the checklist first, then model the categories, filters, and check items around that process.

## Best Practices

- Design categories around real stages of work, not around technical configuration.
- Keep template headers object-specific and clearly named.
- Use filters only where they reduce noise for end users.
- Keep custom answer options tightly controlled and easy to interpret in reporting.
- Use flow launching sparingly for high-value follow-up actions after completion.

## Avoid

- Building very large all-purpose checklists that appear on every record.
- Creating overlapping template headers that are hard for admins to troubleshoot.
- Using check comments as the main store for important operational data.
- Launching flows for routine actions unless there is a clear business need.
